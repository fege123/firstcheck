Git ו-GitHub: המדריך המלא לפיתוח מושלם

מדריך מקיף וחדשני להתקנה, הגדרה ושימוש ב-Git וב-GitHub.

מה זה Git?

Git היא מערכת חכמה לניהול גרסאות (Version Control System). היא מאפשרת לצוותים לעבוד על אותו פרויקט, לעקוב אחר שינויים ולנהל גרסאות בצורה חכמה. זהו כלי חובה לכל מפתח. Git עוקבת אחר שינויים בקבצים ומסמכים והיא מהווה מעין מאגר (Repository) העוקב אחר שינויים בקבצי הקוד של הפרויקט שלנו.

Git מאפשרת לנו לעבוד בצוות, כל אחד על חלקו ולשגר את השינויים למאגר אחד המכיל את כל השינויים של כולם. שמו המקצועי של מאגר זה הוא "Repository".

כל שינוי המבוצע ומעודכן במאגר נקרא "Commit". כל Commit כולל תיאור קצר המפרט על השינוי וכן את שם המפתח האחראי.

היתרונות בשימוש ב-Git

עבודה מסודרת בצוות

השימוש ב-Git עוזר לעבוד כיחיד ובעיקר כצוות באופן מסודר וטוב יותר.

חזרה מהירה לגרסאות קודמות

מאפשר לחזור לגרסאות ישנות במהירות ובפשטות.

גיבוי חכם ומאובטח

מהווה גיבוי חכם עם תיעוד גרסאות לקבצי הפרויקט.

שלושת האזורים העיקריים של Git

Working Copy

קבצי הפרויקט על המחשב המקומי. קבצים ששונו ולא הוספו למעקב יופיעו בצבע אדום.

Staging Copy

אזור המתנה לקבצים שהוספתם למעקב באמצעות git add. קבצים אלו יופיעו בצבע ירוק.

Committed Copy

הקבצים שאושרה עליהם שמירה (Commit). השינוי נדחף למאגר והופך לחלק מההיסטוריה.

התקנה והגדרה ראשונית

  1. התקנת Git

    הורידו גרסה מתאימה מהאתר הרשמי: git-scm.com/download/win. לאחר ההתקנה, הפעילו את git-bash.exe וודאו את הגרסה המותקנת.

    git --version

  2. הגדרת המשתמש האישי

    לכל "חבר" במאגר יש שם משתמש וכתובת מייל המוגדרים ב-Git כדי לדעת מי אחראי ל-Commit מסוים.

    git config --global user.name "Your Name"
git config --global user.email "youremail@example.com"
// בדיקה שהפרטים נקלטו:
git config --list

פקודות Git עיקריות (מקומי)

יצירת מאגר (Repository) והוספת קבצים

כדי להתחיל לנהל את הפרויקט, עליכם להפוך את התיקייה למאגר Git.

// בצעו זאת בתיקיית הפרויקט הרצויה.
git init
// לאחר מכן, הוסיפו את כל הקבצים למעקב:
git add .

שמירת שינויים (Commit)

בצעו שמירה של השינויים עם תיאור מפורט. שימו לב: לא מומלץ להשתמש ב-git commit -am כיוון שאינו מאפשר חזרה לאחור.

git commit -m "תיאור השינוי שבוצע"

ניהול קבצים

// בדיקת סטטוס הקבצים
git status
// ביטול הוספה של קובץ לאזור ה-Staging
git rm --cached [שם-קובץ]
// מחיקת קובץ
git rm [שם-קובץ]
// העברת קובץ
git mv [שם-קובץ] [תיקיית-יעד/שם-קובץ]

פקודות מתקדמות

קובץ .gitignore

קובץ המורה ל-Git להתעלם מקבצים או תיקיות ספציפיות שלא תרצו לכלול במאגר. לדוגמה, קבצי יומן או קבצי הרצה.

// יצירת הקובץ .gitignore
touch .gitignore
// לאחר יצירתו, יש לערוך אותו ולהוסיף שמות של קבצים/תיקיות להורדה.
// לדוגמה, בתוך הקובץ:
myproject.pyc
*.log
/temp/
// מומלץ להוסיף את הקובץ למאגר:
git add .gitignore

צפייה בהיסטוריה והשוואת גרסאות

// צפייה ב-commit-ים קודמים
git log
// השוואה בין ה-Working Copy ל-Repository
git diff
// השוואה בין ה-Working Copy ל-Staging Area
git diff --staged

עבודה עם GitHub

GitHub הוא שירות ענן המהווה מעין רשת חברתית ל-Git. הוא מאפשר לשתף ולנהל את הפרויקטים בענן.

  1. הגדרת מאגר מרוחק (Remote)

    לאחר יצירת מאגר ב-GitHub, העתיקו את ה-URL שלו וחברו אותו למאגר המקומי שלכם. origin הוא הכינוי המקובל למאגר המרוחק. 

    git remote add origin https://github.com/YourUserName/your-repo.git

  2. שליחה וקבלת עדכונים מהמאגר המרוחק

    פקודת Push: נועדה לדחוף שינויים מהמחשב המקומי לשרת המרוחק. 

    git push -u origin main

    פקודת Pull: נועדה למשוך שינויים מהשרת המרוחק למחשב המקומי. 

    git pull origin main

  3. יצירת עותק מקומי ממאגר קיים (Clone)

    פקודה זו יוצרת עותק של מאגר מרוחק על המחשב שלכם, כולל כל ההיסטוריה שלו.

    git clone https://github.com/twbs/bootstrap.git

ענפים (Branches)

ענף הוא "גרסה" נפרדת של המאגר שלכם. main הוא שם ברירת המחדל לענף הראשי. יצירת ענפים מאפשרת עבודה על תכונות חדשות או תיקונים מבלי לפגוע בגרסה היציבה. 

// יצירת ענף חדש
git branch newbranch
// מעבר לענף חדש
git checkout newbranch
// דרך קצרה: יצירת ענף ומעבר אליו בפקודה אחת
git checkout -b version2
// צפייה בכל הענפים הקיימים
git branch
// מחיקת ענף
git branch -D version2

אינטגרציה עם סביבות פיתוח

סביבות פיתוח כמו PyCharm ו-Android Studio מציעות תמיכה מובנית מלאה ב-Git, כך שאין צורך להשתמש בטרמינל.

PyCharm

הגדרת GitHub: File > Settings > Version Control > GitHub.
פתיחת פרויקט: VCS > Checkout from Version Control > Git.

Android Studio

הגדרת GitHub: File > Settings > Version Control > GitHub.
פתיחת פרויקט: VCS > Get from Version Control.

טיפ: לאחר הגדרה, כל הפעולות כמו Commit, Push ו-Pull זמינות דרך התפריטים השונים של סביבת הפיתוח.

תרגיל לתלמידים

כעת הגיע הזמן ליישם את מה שלמדתם:

  1. התחברות למאגר

    התחברו למאגר (repository) שלנו ב-GitHub.

  2. יצירת קובץ חדש

    צרו קובץ חדש בשם שלכם.

  3. הוספת תוכן לקובץ

    הוסיפו תוכן כלשהו לקובץ, למשל את שמכם.

  4. שמירת השינוי

    בצעו Commit כדי לשמור את השינוי במאגר המקומי שלכם.

מטרה: בסיום התרגיל, כל תלמיד אמור לראות את הקובץ שלו במאגר המקומי וב-GitHub.

סיכום

ניהול תצורה יעיל באמצעות Git ו-GitHub מאפשר:

  • שליטה מלאה בכל תהליך השינויים בקוד.
  • זיהוי חד-ערכי וברור של גרסאות, קשרים ופרטי תצורה.
  • ניהול גרסאות והיסטוריה לכל סוג קובץ.
  • יכולת בנייה ושחזור של כל גרסה.
  • מעקב אחר קשרים בין רכיבי תוכנה.
  • יכולת דיווח והעברת מידע שוטפת בין כל הגורמים.
  • חזרה לגרסה קודמת של המוצר תוך ביצוע שינויים נדרשים.
